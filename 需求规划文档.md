# 学生意见反馈系统 - 需求规划文档

## 一、项目背景与目标

### 1.1 项目背景
加强教学日常管理，学生的反馈是其中重要一个环节，需要给学生建立反馈的通道，让学生可以随时对教学、管理等方面提出意见和建议。

### 1.2 项目目标
- 建立学生与学校管理层之间的沟通渠道
- 保护学生隐私，支持匿名反馈
- 及时发现和解决教学管理中的问题
- 为学校管理决策提供数据支持

### 1.3 核心原则
- **隐私保护**: 反馈信息只有特定管理员可查看，教师无法直接看到
- **及时响应**: 支持随时反馈，及时处理
- **权限分级**: 不同类别的反馈由不同管理员负责
- **数据安全**: 软件公司也无法查看反馈数据

## 二、用户角色定义

### 2.1 学生 (Student)
**权限:**
- 提交反馈（针对本学期任课教师或学校事务）
- 选择匿名或实名
- 查看自己的反馈记录
- 查看管理员的回复
- 补充反馈信息

**限制:**
- 只能对本学期任课教师提出意见
- 无法查看其他学生的反馈

### 2.2 类别管理员 (Category Admin)
**权限:**
- 查看被授权类别的反馈
- 回复学生反馈
- 添加备注提醒（向上级或下级）
- 查看统计数据

**限制:**
- 只能查看被授权的类别反馈
- 无法修改系统配置

### 2.3 系统管理员 (System Admin)
**权限:**
- 所有类别管理员的权限
- 配置反馈类别
- 设置类别权限（分配给特定管理员）
- 查看全局统计数据
- 用户管理

## 三、功能需求

### 3.1 学生端功能

#### 3.1.1 我要反馈
**页面:** 1-1学生首页-我要反馈.html

**功能描述:**
学生可以提交反馈意见，包括对任课教师的意见或学校其他事务的反馈。

**字段要求:**

| 字段 | 类型 | 必填 | 长度限制 | 说明 |
|------|------|------|----------|------|
| 反馈类别 | 下拉选择 | 是 | - | 从系统配置的类别中选择 |
| 是否与任课教师相关 | 单选 | 条件显示 | - | 当类别关联任课教师时显示，选项：是/否 |
| 反馈对象 | 下拉选择 | 条件必填 | - | 当"是否与任课教师相关"=是时必填 |
| 反馈主题 | 文本输入 | 是 | 20字符 | 支持汉字、字母、数字、特殊符号 |
| 反馈内容 | 多行文本 | 是 | 500字符 | 支持汉字、字母、数字、特殊符号 |
| 是否匿名 | 单选 | 是 | - | 默认实名，可选匿名 |
| 附件 | 文件上传 | 否 | - | 支持图片、文档等 |

**业务规则:**
1. 选择反馈类别后，如果该类别"关联任课教师"=是，则显示"是否与任课教师相关"字段
2. 当学生选择"是否与任课教师相关"=是时，显示"反馈对象"下拉框，必须选择具体教师
3. 当学生选择"是否与任课教师相关"=否时，隐藏"反馈对象"字段，反馈将作为该类别的一般性意见
4. 反馈对象列表只显示本学期任课教师
5. 如果选择的类别"关联任课教师"=否，则不显示"是否与任课教师相关"和"反馈对象"字段
6. 提交前进行字段校验
7. 支持暂存草稿功能
8. 提交成功后显示提示信息

**提示语:**
"您的反馈只有学校领导可以查看，请放心提出您的意见和建议"

**使用场景说明:**

**场景1：针对具体教师的反馈**
- 学生选择"教学质量"类别（该类别关联任课教师）
- 系统显示"是否与任课教师相关"选项
- 学生选择"是"
- 系统显示反馈对象下拉框，学生选择"张老师-数学"
- 提交后，该反馈会关联到张老师
- **适用于**: 对某位教师的教学方法、课堂管理等具体意见

**场景2：针对类别的一般性反馈**
- 学生选择"教学质量"类别（该类别关联任课教师）
- 系统显示"是否与任课教师相关"选项
- 学生选择"否"（例如：对整体教学方法的建议，不针对某个教师）
- 系统隐藏反馈对象字段
- 提交后，该反馈作为"教学质量"类别的一般性意见
- **适用于**: 对整体教学安排、课程设置等宏观层面的建议

**场景3：不关联教师的类别**
- 学生选择"食堂服务"类别（该类别不关联任课教师）
- 系统不显示"是否与任课教师相关"和"反馈对象"字段
- 学生直接填写反馈内容并提交
- **适用于**: 后勤服务、设施设备等非教学类反馈

**字段显示逻辑流程图:**
```
选择反馈类别
    ↓
类别是否关联任课教师？
    ↓
是 → 显示"是否与任课教师相关"
    ↓
    学生选择"是" → 显示"反馈对象"（必填）
    学生选择"否" → 隐藏"反馈对象"
    ↓
否 → 不显示"是否与任课教师相关"和"反馈对象"
```

#### 3.1.2 反馈跟踪
**页面:** 1-2反馈跟踪.html

**功能描述:**
学生可以查看自己提交的所有反馈记录，以及管理员的回复。

**列表字段:**
- 反馈编号
- 反馈类别
- 反馈主题
- 提交时间
- 状态（待回复/已回复/已关闭）
- 操作（查看详情）

**详情页显示:**
- 反馈基本信息
- 反馈内容
- 附件（如有）
- 回复记录（时间倒序）
- 补充信息功能

**状态说明:**
- **待回复**: 已提交，等待管理员处理
- **已回复**: 管理员已回复
- **已关闭**: 问题已解决，反馈关闭

### 3.2 管理员端功能

#### 3.2.1 反馈查看
**页面:** 2-1查看管理员首页—反馈查看.html

**功能描述:**
管理员查看、筛选、回复学生反馈。

**筛选条件:**
- 反馈类别（只显示有权限的类别）
- 反馈状态（全部/待回复/已回复/已关闭）
- 时间范围
- 关键词搜索（主题、内容）
- 反馈对象（教师）

**列表字段:**
- 反馈编号
- 反馈类别
- 反馈主题
- 反馈对象
- 提交人（匿名显示"匿名用户"）
- 提交时间
- 状态
- 操作（查看详情、回复、备注提醒）

**详情页功能:**
1. 查看完整反馈信息
2. 查看学生身份（即使是匿名）
3. 回复反馈（支持多次回复）
4. 添加备注提醒
5. 更改状态
6. 查看处理历史

**回复功能:**
- 回复内容：多行文本，500字符
- 回复后学生可见
- 支持追加回复

#### 3.2.2 备注提醒
**页面:** 2-1-5备注提醒.html

**功能描述:**
管理员可以对某条反馈添加备注，并提醒其他管理员关注。

**字段:**
- 提醒对象：选择其他管理员
- 备注内容：500字符
- 紧急程度：普通/重要/紧急

**提醒方式:**
- 系统消息通知
- 被提醒人登录后在消息中心查看

#### 3.2.3 反馈统计
**页面:** 2-2反馈统计.html

**功能描述:**
统计分析反馈数据，为管理决策提供支持。

**统计维度:**
1. **按类别统计**
   - 各类别反馈数量
   - 各类别处理率
   - 各类别平均处理时长

2. **按教师统计**
   - 被反馈次数排名（TOP 10）
   - 各教师被反馈的类别分布
   - 点击可查看详情

3. **按时间统计**
   - 每日/每周/每月反馈趋势
   - 处理效率趋势

4. **按状态统计**
   - 待处理数量
   - 已处理数量
   - 平均处理时长

**导出功能:**
- 支持导出统计报表（Excel）
- 支持导出反馈详情列表

### 3.3 系统管理功能

#### 3.3.1 反馈类别管理
**页面:** 2-3反馈管理.html

**功能描述:**
系统管理员配置反馈类别。

**类别字段:**
- 类别名称：10字符，汉字、字母、数字
- 是否关联任课教师：是/否
- 显示顺序：数字，控制前端显示顺序
- 状态：启用/禁用

**操作:**
- 新增类别
- 编辑类别
- 删除类别（需确认）
- 排序

**预置类别示例:**
- 教学质量（关联教师）
- 课堂纪律（关联教师）
- 教学设施
- 食堂服务
- 宿舍管理
- 其他建议

#### 3.3.2 权限配置
**页面:** 3-2类别权限配置.html

**功能描述:**
为管理员分配可查看的反馈类别权限。

**配置方式:**
1. 选择管理员
2. 勾选可查看的类别
3. 保存配置

**权限规则:**
- 系统管理员默认拥有所有类别权限

**页面:** 3-1角色权限设置.html

**功能描述:**

**角色定义:**
- 学生：提交反馈、查看自己的反馈

## 四、非功能需求

### 4.1 性能要求
- 页面响应时间 < 2秒
- 支持并发用户数 > 1000
- 数据库查询优化，列表分页加载

### 4.2 安全要求
- 用户密码加密存储（BCrypt）
- JWT Token 认证
- 接口权限校验
- 防止 SQL 注入、XSS 攻击
- 敏感数据加密传输（HTTPS）
- 操作日志记录

### 4.3 数据隐私
- 匿名反馈的学生身份只有管理员可见
- 反馈内容只有授权管理员可查看
- 软件公司无法访问生产数据
- 定期数据备份

### 4.4 兼容性
- 浏览器：Chrome、Firefox、Edge、Safari 最新版本
- 移动端：响应式设计，支持手机、平板访问
- 分辨率：1920x1080、1366x768、移动端适配

### 4.5 可用性
- 界面简洁友好
- 操作流程清晰
- 错误提示明确
- 支持快捷键操作

## 六、业务流程

### 6.1 学生提交反馈流程
```
1. 学生登录系统
2. 点击"我要反馈"
3. 选择反馈类别
4. 如果类别"关联任课教师"=是，显示"是否与任课教师相关"选项
   4.1 如果学生选择"是"，显示反馈对象下拉框（只显示本学期任课教师）
   4.2 如果学生选择"否"，隐藏反馈对象字段，作为该类别的一般性反馈
5. 填写反馈主题和内容
6. 选择是否匿名
7. 上传附件（可选）
8. 点击提交
9. 系统校验数据
10. 保存到数据库
11. 显示提交成功提示
12. 学生可在"反馈跟踪"中查看
```

### 6.2 管理员处理反馈流程
```
1. 管理员登录系统
2. 进入"反馈查看"页面
3. 系统根据权限过滤显示反馈列表
4. 管理员筛选、搜索反馈
5. 点击查看详情
6. 阅读反馈内容
7. 填写回复内容
8. 提交回复
9. 系统保存回复
10. 学生可在"反馈跟踪"中看到回复
11. 管理员可选择关闭反馈
```

### 6.3 备注提醒流程
```
1. 管理员A查看某条反馈
2. 认为需要其他管理员关注
3. 点击"备注提醒"
4. 选择提醒对象（管理员B）
5. 填写备注内容
6. 选择紧急程度
7. 提交
8. 系统发送消息通知给管理员B
9. 管理员B登录后在消息中心看到提醒
10. 点击查看相关反馈
```

- GET `/api/system/categories` - 类别列表

- PUT `/api/system/categories/{id}` - 更新类别
- DELETE `/api/system/categories/{id}` - 删除类别
- PUT `/api/system/categories/order` - 更新排序
- GET `/api/system/admins` - 管理员列表
- GET `/api/system/permissions` - 权限配置列表
- POST `/api/system/permissions` - 配置权限
- DELETE `/api/system/permissions/{id}` - 删除权限配置

## 八、技术实现要点

### 8.1 任课关系验证
- 从课表系统同步学生-教师关联数据
- 提交反馈时验证教师是否为学生本学期任课教师
- 每学期初更新任课关系数据

### 8.2 权限控制
- 基于 Spring Security + JWT 实现认证
- 自定义权限注解 `@RequirePermission`
- 拦截器验证用户是否有权限查看该类别反馈
- 前端根据用户角色动态显示菜单

### 8.3 匿名处理
- 数据库存储真实学生ID
- 查询时根据用户角色决定是否显示真实姓名
- 管理员可查看匿名学生真实身份
- 前端显示时用"匿名用户"替代

### 8.4 消息通知
- 使用 Redis 存储未读消息计数
- WebSocket 实时推送新消息
- 定时任务清理过期消息

### 8.5 数据统计
- 使用 Redis 缓存统计结果（5分钟过期）
- 定时任务预计算统计数据
- 大数据量导出使用异步任务

## 十、风险与应对

### 10.1 技术风险
**风险:** 并发量大时性能下降
**应对:**
- 使用 Redis 缓存
- 数据库索引优化
- 分页加载
- 负载均衡

### 10.2 数据安全风险
**风险:** 敏感数据泄露
**应对:**
- 数据加密存储和传输
- 严格的权限控制
- 操作日志审计
- 定期安全扫描

### 10.3 业务风险
**风险:** 恶意反馈或刷屏
**应对:**
- 限制提交频率（每天最多5条）
- 敏感词过滤
- 管理员可标记和删除恶意反馈
- 记录操作日志

### 10.4 用户体验风险
**风险:** 用户不会使用或不愿使用
**应对:**
- 简化操作流程
- 提供操作指引
- 用户培训
- 收集反馈持续优化
